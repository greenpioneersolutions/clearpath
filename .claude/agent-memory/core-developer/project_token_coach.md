Token Coach is a 5-phase initiative to make ClearPath the meter between users and the CLI vendors, ahead of the June 2026 shift to usage-based billing across Copilot CLI / Claude Code / Codex.

**Why:** On June 1 2026 GitHub Copilot CLI moves to AI-credit billing; June 15 Anthropic does the same for Claude Code; OpenAI already did April 2026. ClearPath sees every outbound request and can measure + rewrite before it ships.

**How to apply:** Each phase plugs into the same middleware pipeline at `src/main/cli/middleware/`. Future phases (3-5) should slot new middlewares into `defaultPipeline` between `lintMiddleware` and `measureMiddleware` — measure must always run last so its breakdown reflects the final shape of the prompt.

Phase status:
- **Phase 1 (Truth)**: ✅ merged. `tokenCounter` singleton in [src/main/tokenization/TokenCounter.ts](src/main/tokenization/TokenCounter.ts); CostRecord has optional slice fields; `cost:turn-breakdown` IPC.
- **Phase 2 (Visibility)**: ✅ merged. Middleware pipeline (`src/main/cli/middleware/`); `cli:prompt-shaped` IPC event; `tokenizer:count-multi` IPC; `showTokenMeter` flag (default ON); `ContextMeterChip` + popover in chat input.
- **Phase 3 (Stability)**: ✅ implemented (working tree, not yet merged). `prefixOrderMiddleware` between lint and measure; canonical order is `fleet → agent → notes → contextSources → userText`. `cachePolicy.ts` with per-model minimums (1024 default, 4096 for Opus 4.5+/Haiku 4.5). `LocalModelAdapter.chatAnthropic` injects `cache_control` when policy enabled + Anthropic model + prefix ≥ min. `cli:prompt-shaped` payload extended with optional `cacheStatus`. `notes:get-bundle-for-prompt` sorts notes by id and attachments by filename. `showPromptCache` flag (experimental, default OFF). Settings: `settings:get-cache-policy` / `settings:set-cache-policy` with `onCachePolicyChange` listener fan-out.
- **Phase 4 (Routing)**: ✅ implemented (working tree, not yet merged). Pure-heuristic `DifficultyClassifier` (`src/main/routing/DifficultyClassifier.ts`) returns `trivial | normal | hard` + confidence + reasons. Per-CLI `RoutingRules` stored on AppSettings (`settings:get-routing-rules` / `settings:set-routing-rules` + `onRoutingRulesChange` fan-out). `routingMiddleware` slots after measure via `buildPipeline({ routing: { getRules } })` — CLIManager constructs the pipeline once with a getter closure so live rules updates take effect on next turn. `routing:classify` + `routing:resolve-tier` IPCs power the renderer chip. `ModelRoutingChip` shows the routed model + tier buttons + reasons tooltip; flag-gated by `showModelRouting` (experimental, default OFF). CostRecord extended with `routedModel`, `userOverride`, `routedDifficulty`. `cli:prompt-shaped` payload extended with optional `routing: { routedModel, userOverride, difficulty, reasons, confidence }`.
- **Phase 5 (Feeling)**: ✅ implemented (working tree, not yet merged). `warningMiddleware` runs LAST in pipeline via `buildPipeline({ warning: { getPricingTable } })` — emits one severity-prefixed note per turn into `ctx.notes` (rank: context-window > high-cost > huge-attachment > cheap-route-override). Renderer surfaces: `PreflightWarningStack` subscribes to `cli:prompt-shaped.notes` and shows up to 2 banners above chat input, auto-dismissing on textarea edit (driven by `editTick` from `ChatInputArea.onTextChange`). `CompactNudge` watches running session tokens vs `contextWindowFor(model)` and shows a 70% nudge with a "Compact now" CTA that dispatches `/compact`. "Fresh start" button in session header spawns a new session with the same agent + model. Insights → Efficiency tab (`src/renderer/src/pages/insights/EfficiencyTab.tsx`) renders donut of token slices, top-context-bloat table, optional routing distribution pie (when `showModelRouting` is also on), and real-data savings cards. 3 new IPC handlers in `efficiencyHandlers.ts`: `efficiency:where-did-tokens-go` / `:top-context-bloat` / `:savings-suggestions` — all read-only aggregations from the existing cost + session stores. `showEfficiencyInsights` flag (experimental, default OFF).
